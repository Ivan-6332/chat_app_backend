# API Documentation

## Base URL
```
http://localhost:8080/api/v1
```

## Authentication
All endpoints (except `/health`) require a valid JWT token from Auth0 in the Authorization header:
```
Authorization: Bearer <your-jwt-token>
```

---

## Endpoints

### Health Check
Check if the service is running.

**Endpoint:** `GET /health`

**Auth Required:** No

**Response:**
```json
{
  "success": true,
  "data": {
    "status": "healthy",
    "service": "chatapp-backend"
  },
  "message": "Service is running"
}
```

---

### Send Message
Send an encrypted message to a conversation.

**Endpoint:** `POST /api/v1/messages`

**Auth Required:** Yes

**Request Body:**
```json
{
  "senderId": "string (required) - ID of the message sender",
  "conversationId": "string (required) - ID of the conversation",
  "encryptedText": "string (required) - Encrypted message content"
}
```

**Success Response (201 Created):**
```json
{
  "success": true,
  "data": {
    "id": "generated-uuid",
    "senderId": "user123",
    "conversationId": "conv456",
    "encryptedText": "encrypted_content",
    "timestamp": "2026-03-08T10:30:00Z"
  },
  "message": "Message sent successfully"
}
```

**Error Response (400 Bad Request):**
```json
{
  "success": false,
  "error": "Invalid request body: senderId is required"
}
```

**Error Response (401 Unauthorized):**
```json
{
  "success": false,
  "error": "Authorization header is required"
}
```

### Get Messages by Conversation
Retrieve all messages for a specific conversation.

**Endpoint:** `GET /api/v1/messages/:conversationId`

**Auth Required:** Yes

**URL Parameters:**
- `conversationId` (string, required) - The ID of the conversation

**Success Response (200 OK):**
```json
{
  "success": true,
  "data": [
    {
      "id": "msg-uuid-1",
      "senderId": "user123",
      "conversationId": "conv456",
      "encryptedText": "encrypted_content_1",
      "timestamp": "2026-03-08T10:30:00Z"
    },
    {
      "id": "msg-uuid-2",
      "senderId": "user456",
      "conversationId": "conv456",
      "encryptedText": "encrypted_content_2",
      "timestamp": "2026-03-08T10:35:00Z"
    }
  ],
  "message": "Messages retrieved successfully"
}
```

**Empty Response (200 OK):**
```json
{
  "success": true,
  "data": [],
  "message": "Messages retrieved successfully"
}
```

---

### Get Conversations by User
Retrieve all conversations for a specific user.

**Endpoint:** `GET /api/v1/conversations/:userId`

**Auth Required:** Yes

**URL Parameters:**
- `userId` (string, required) - The ID of the user

**Success Response (200 OK):**
```json
{
  "success": true,
  "data": [
    {
      "id": "conv456",
      "participants": ["user123", "user456"],
      "lastMessage": "encrypted_last_message",
      "lastTimestamp": "2026-03-08T10:35:00Z",
      "createdAt": "2026-03-08T09:00:00Z"
    },
    {
      "id": "conv789",
      "participants": ["user123", "user789"],
      "lastMessage": "encrypted_last_message_2",
      "lastTimestamp": "2026-03-08T11:00:00Z",
      "createdAt": "2026-03-08T08:00:00Z"
    }
  ],
  "message": "Conversations retrieved successfully"
}
```

## Error Codes

| Status Code | Description |
|-------------|-------------|
| 200 | Success |
| 201 | Created |
| 400 | Bad Request - Invalid input |
| 401 | Unauthorized - Missing or invalid token |
| 500 | Internal Server Error |

All errors follow this format:
```json
{
  "success": false,
  "error": "Error message describing what went wrong"
}
```

## Data Models

### Message
```typescript
{
  id: string;           // UUID generated by server
  senderId: string;     // Auth0 user ID
  conversationId: string;
  encryptedText: string; // End-to-end encrypted message
  timestamp: DateTime;   // ISO 8601 format
}
```

### Conversation
```typescript
{
  id: string;              // Unique conversation identifier
  participants: string[];  // Array of user IDs
  lastMessage?: string;    // Last encrypted message (optional)
  lastTimestamp: DateTime; // When last message was sent
  createdAt: DateTime;     // When conversation was created
}
```

---

## Security Notes

1. **No Decryption**: Backend never decrypts messages - it only stores and retrieves encrypted data
2. **JWT Validation**: All requests are validated against Auth0's JWKS
3. **Token Caching**: JWKS keys are cached for 1 hour to reduce external calls
4. **Audience Verification**: Tokens must match the configured API audience
5. **Issuer Verification**: Tokens must be issued by the configured Auth0 domain

## Example Usage with cURL

**Send a message:**
```bash
curl -X POST http://localhost:8080/api/v1/messages \
  -H "Authorization: Bearer eyJhbGc..." \
  -H "Content-Type: application/json" \
  -d '{
    "senderId": "auth0|123456",
    "conversationId": "conv-abc-123",
    "encryptedText": "U2FsdGVkX1+..."
  }'
```

**Get messages:**
```bash
curl -X GET http://localhost:8080/api/v1/messages/conv-abc-123 \
  -H "Authorization: Bearer eyJhbGc..."
```

**Get conversations:**
```bash
curl -X GET http://localhost:8080/api/v1/conversations/auth0|123456 \
  -H "Authorization: Bearer eyJhbGc..."
```
